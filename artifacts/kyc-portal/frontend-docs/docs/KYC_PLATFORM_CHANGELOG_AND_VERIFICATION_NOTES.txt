KYC_PLATFORM_CHANGELOG_AND_VERIFICATION_NOTES
Date: 2025-12-25

Overview
- Added API-key based authentication, environment gating, and scope enforcement for all FastAPI agent endpoints (`kyc-agents-main/kyc-agents-main/common/auth.py`, `common/http.py`), including structured error codes and metrics. Auth payloads are now seeded from `AuthContext` prior to request normalization and a shared ApiKeyStore instance is reused per process.
- Introduced client-side artifact encryption + per-tenant/subject key hierarchy with crypto-shredding hooks (`common/encryption.py`, `common/key_management.py`, `common/artifacts.py`), plus retention job primitives and dry-run capable cleanup (`common/retention.py`, `cdktf/src/backend/retention/job.ts`). Retention scans now paginate through all DynamoDB pages with throttling backoff.
- Provisioned new DynamoDB tables, IAM permissions, KMS key, and a scheduled retention Lambda in infrastructure (`cdktf/src/constructs/storage/index.ts`, `cdktf/src/stacks/dev.ts`, `cdktf/src/constructs/compute/index.ts`). The retention Lambda now runs with reserved concurrency = 1 and alarms notify `${prefix}-alerts`.
- Added detailed tests for auth parsing, scope/IP enforcement, encryption/retention flows (`kyc-agents-main/kyc-agents-main/tests/test_auth.py`, `tests/test_crypto_retention.py`) plus new cases covering auth injection and scan pagination.
- Enforced Edge AuthN/AuthZ at API Gateway (HTTP API Lambda authorizer with simple response) using the same API key table + Argon2 verification as the agents (`cdktf/src/backend/auth/authorizer.ts`, `cdktf/src/stacks/dev.ts`). All HTTP API routes require `Authorization: Bearer api_key_id.secret`; environment is now server-authoritative (derived only from the API key record) and `x-env`/`x-tenant-id` are not used.
- Enforced Admission Guard for costly work: `POST /jobs` runs the TypeScript admission checks (strongly consistent reads) before starting Step Functions (`cdktf/src/backend/admission/index.ts`, `cdktf/src/backend/controller/index.ts`).
- Wired rating events into the deployed Billing Lambda via SQS -> Lambda event source mapping (FIFO queue + DLQ) and added SQS batch partial failure handling in the billing handler (`cdktf/src/constructs/billing/index.ts`, `cdktf/src/backend/billing/index.ts`).
- Standardized production agent execution to SQS -> ECS workers (FastAPI `/process` remains for local/dev). Agent containers now default to a long-poll SQS worker loop (`kyc-agents-main/kyc-agents-main/common/sqs_worker.py`, `*/main.py`, `cdktf/src/constructs/compute/index.ts`).
- Hardened artifacts S3 bucket: block public access, enforce TLS, and enforce SSE-S3 (`AES256`) via default encryption + bucket policy; updated all writers/presign flows to include the required SSE header (`cdktf/src/constructs/storage/index.ts`, `cdktf/src/backend/controller/index.ts`, `cdktf/src/backend/video/submit.ts`, `kyc-agents-main/kyc-agents-main/common/s3.py`, `kyc-agents-main/kyc-agents-main/common/artifacts.py`).
- Reduced EMF metric cardinality by removing `tenant_id` as a dimension in agents metrics (`kyc-agents-main/kyc-agents-main/common/metrics.py`).
- Added an Edge “front door” for the HTTP API using CloudFront + WAFv2 (CLOUDFRONT scope), with an origin verification secret header to prevent direct API Gateway invoke URL bypass (`cdktf/src/stacks/dev.ts`, `cdktf/src/backend/auth/authorizer.ts`). The origin secret is stored as an SSM SecureString and injected into both CloudFront and the authorizer.
- Upgraded the deployed Billing Lambda to “v2” rating: rate rules come from a DynamoDB RateCard table (with minimal DEFAULT ratecards seeded at deploy time) and pricing comes from PlanSnapshot + TenantOverrides; writes are now fully atomic via DynamoDB TransactWriteItems (idempotency marker + ledger + balance + daily usage in a single transaction) (`cdktf/src/backend/billing/index.ts`, `cdktf/src/constructs/billing/index.ts`).
- Enabled SANDBOX usage tracking via rating events (SANDBOX remains $0, but now increments sandbox daily usage counters used by Admission Guard) (`kyc-agents-main/kyc-agents-main/common/events.py`, `cdktf/src/backend/billing/index.ts`, `cdktf/src/backend/admission/index.ts`).
- Added a minimal control-plane skeleton (admin-gated by `admin:write` + `x-admin-token`) including tenant onboarding (`POST /tenants`), key management (`POST /apikeys`, `POST /apikeys/rotate`, `POST /apikeys/revoke`, `GET /apikeys/{apiKeyId}`), billing topups (`POST /billing/topup`), KYB lifecycle (`POST /kyb/start`, `POST /kyb/complete`), risk tier (`POST /risk/recompute`, `POST /risk/override`), notifications (`POST /notify` to an SQS bus), and plan management (`POST /plans/snapshot`, `POST /plans/overrides`). All admin actions write durable entries to the admin-audit table (`cdktf/src/backend/controller/index.ts`, `cdktf/src/stacks/dev.ts`).
- Expanded observability: ops dashboard + alarms now include edge/API latency (p50/p99), edge/API 4xx/5xx signals, WAF request counters, authorizer duration (p99), DynamoDB throttle alarms, and additional queue age alarms (`cdktf/src/stacks/dev.ts`, `cdktf/src/backend/auth/authorizer.ts`).
- Implemented a real daily Penalty Engine sweep: promotes `ON_CREDIT -> OVERDUE` after `grace_period_days`, applies daily penalties while `OVERDUE` (bounded by `penalty_cap_ratio` and `penalty_rate_per_day`), and optionally transitions `OVERDUE -> LOCKED` after `lock_after_overdue_days` (`cdktf/src/backend/billing/index.ts`, `cdktf/src/constructs/billing/index.ts`).

Rationale
- Harden public API access with API keys, Argon2id hashing, audit logging, scope checks, environment isolation, IP allowlists, and metrics for observability and abuse response.
- Meet data retention and crypto-shredding requirements by encrypting artifacts per subject, minimizing KMS calls via tenant master keys, and adding automated cleanup with legal hold awareness and dry-run safety.
- Extend infra with least-privilege tables/roles and scheduled retention processing to support compliance and operational runbooks.

Latest Iteration Addenda
- Controller identity is now derived exclusively from HTTP API authorizer context (`event.requestContext.authorizer.lambda.*`). Any request missing authorizer context is treated as unauthenticated (401), and caller-provided tenant/env headers are ignored.
- Admission Guard emits low-cardinality metrics (no `tenant_id` dimension) while still logging tenant details in structured logs for debugging (`cdktf/src/backend/admission/index.ts`).
- The Step Functions -> agent SQS integration now uses `States.JsonToString($)` for `MessageBody` (SQS requires string bodies) and includes server-authoritative `tenantId` and `env` in the message payload (`cdktf/src/stacks/dev.ts`). Agents parse JSON and construct requests from those fields.
- SFN work items now include `request_id`/`usage_id` (deterministic per job+tool+env), plus `subject_id` and `data_class` when present. SQS FIFO deduplication uses `request_id` to avoid double-enqueue on SFN retries (`cdktf/src/backend/controller/index.ts`, `cdktf/src/stacks/dev.ts`, `kyc-agents-main/kyc-agents-main/common/models.py`).
- FIFO note: SQS FIFO deduplication is only guaranteed within the 5-minute content-based window; downstream idempotency still applies (agents must be safe under reprocessing, and billing uses a DynamoDB idempotency marker keyed by `usage_id`).
- Mermaid deviation: `video-liveness` remains `waitForTaskToken` (Lambda/SageMaker path) rather than SQS->ECS worker. This is intentionally preserved to match current deployed behavior and reduce risk; the SQS agent path is used for the other tools (`cdktf/src/stacks/dev.ts`).
- Rating ingestion is now “real”: agents/services emit events to `RATING_EVENTS_QUEUE_URL` and Billing Lambda consumes them via event source mapping with `ReportBatchItemFailures` enabled; malformed messages are retried and eventually DLQ’d without blocking the full batch (`cdktf/src/constructs/billing/index.ts`, `cdktf/src/backend/billing/index.ts`).
- Billing schema drift is fixed in controller endpoints: `TenantBalance` is keyed by `tenant_id` only (no environment key) and ledger reads use `Query` on the tenant PK + time-sorted SK (no fake `GSI_TenantLedger`) (`cdktf/src/backend/controller/index.ts`).
- S3 hardening requires that presigned clients include `x-amz-server-side-encryption: AES256` (SSE-S3). The controller presign endpoint now returns `requiredHeaders` to enforce this on the caller side (`cdktf/src/backend/controller/index.ts`).
- Agent tasks now receive `KYC_JOB_TABLE_NAME` so best-effort retention index pointer writes can happen at artifact write time (reduces reliance on backfill tools) (`cdktf/src/constructs/compute/index.ts`).
- `common/http.py` now seeds payloads with tenant/environment from `AuthContext` before normalization and reuses a module-scoped ApiKeyStore to eliminate per-request boto3 client creation.
- `common/retention.py` and `cdktf/src/backend/retention/job.ts` scan all DynamoDB pages with a simple exponential backoff on throttling to ensure every expired artifact/subject/tenant is evaluated.
- `cdktf/src/stacks/dev.ts` configures the retention Lambda with `reservedConcurrentExecutions=1` and wires the retention error alarm to `${prefix}-alerts` SNS; `cdktf/src/constructs/compute/index.ts` now surfaces that topic ARN for reuse.
- `CachingApiKeyStore` adds a per-process TTL/LRU cache that honors Argon2 expiry/revocation semantics and exposes `API_KEY_CACHE_*` env knobs; `create_agent_app` instantiates a single shared Dynamo store wrapped by the cache to remove per-request boto3 clients.
- DynamoDB retention tables now expose `GSI_Expire`, `GSI_Scrub`, and `GSI_TenantDelete`, and both the Python retention job and the Lambda backend issue bounded `Query` calls (with backoff + LastEvaluatedKey pagination) instead of scans.
- Retention processing enforces `RETENTION_MAX_ITEMS_PER_CATEGORY`, `RETENTION_QUERY_*` limits, and `RETENTION_TIME_BUDGET_SECONDS|MS` deadlines while logging processed/skipped counts per phase.
- Added `tools/backfill_retention_indexes.py` to seed shard/sort attributes for legacy records; run it once per environment before enabling the Query-only retention job.
- Edge authorizer API-key cache: the HTTP API Lambda authorizer now includes an explicit in-process TTL cache with single-flight (stampede protection) for `(api_key_id, secret)` verification. Cache TTL is short (default 10s, max 60s) and is clamped to `expires_at` and (when applicable) the NEXT-secret rotation window. Denies are cached only briefly (default 2s). Metrics are low-cardinality (no tenant/api_key dimensions): `kyc.api_key_cache_hit_total`, `kyc.api_key_cache_miss_total`, `kyc.api_key_cache_singleflight_wait_total`, plus rotation/revocation counters (`cdktf/src/backend/auth/authorizer.ts`).
- Legal hold checks now issue strongly consistent reads against the base `legal-hold` table (scope + tenant_scope) in both Python agents and the retention Lambda, preventing deletion/scrubbing when a hold was just written but not yet visible via GSIs.
- `docs/ONBOARDING_BILLING_COST_LIMITING_ADMISSION_GUARD.md` now uses Markdown-friendly headings, emphasis, and bullets (content unchanged) to improve readability for onboarding/billing reviewers.

Data Model Changes (DynamoDB)
- `api-keys`: pk `api_key_id`, attrs `hashed_secret` (CURRENT), optional `hashed_secret_next` + `next_active_from` + `rotation_grace_seconds` (NEXT rotation), revoke metadata (`revoked_at`, `revoked_by`, `revoke_reason`), plus tenant/env/scopes/status/expires_at (TTL).
- `api-key-audit`: pk `tenant_id`, sk `timestamp#api_key_id#action` (edge authorizer writes `AUTH_ALLOW` / `AUTH_DENY` with request metadata in `details`).
- `subjects`: pk `tenant_id`, sk `subject_id`, attrs earliest_deletion_at, subject_deleted, retention_policy_id, `scrub_shard`=`SCRUB`, `scrub_sort`=`earliest_deletion_at#tenant_id#subject_id` (feeds `GSI_Scrub`).
- `jobs-retention`: pk `tenant_id`, sk `job_id`, attr expires_at (TTL), data_class, s3_key, `expire_shard`=`EXPIRE`, `expire_sort`=`expires_at#tenant_id#job_id` (feeds `GSI_Expire`).
- `subject-keys`: pk `tenant_id`, sk `subject_id`, attrs enc_data_key, key_status.
- `tenant-keys`: pk `tenant_id`, attrs enc_tenant_master_key, status, env_access, `tenant_delete_shard`=`TENANT_DELETE`, `tenant_delete_sort`=`tenant_data_retention_until#tenant_id` (feeds `GSI_TenantDelete`).
- `legal-hold`: pk `scope`, sk `tenant_scope`, GSI_TenantHold on tenant_id/created_at. Retention paths now issue strongly consistent `GetItem` calls on `scope`/`tenant_scope` (no GSI dependency) so newly created holds block destructive work immediately.
- `retention-policies`, `data-requests` created for policy/DSAR tracking (schema documented in code comments).
- Migration helpers: `tools/backfill_retention_indexes.py` scans each table for missing shard/sort attributes and backfills them with the correct composite keys; run it prior to enabling the Query-only retention job and document the change in the rollout checklist.

Goals covered
1) Make `cdktf/src/backend/shared/` real (single source of truth) for SSRF/time/http/json utilities used by running lambdas.
2) Production-hardening improvements:
   - WAF managed rules expansion behind flags
   - key lifecycle operational metrics + dashboard visibility
   - authorizer throttling alarm

What changed (backend)
- SSRF/URL validation is now centralized with typed errors:
  - Added `UrlValidationErrorCode` + `UrlValidationError` in `cdktf/src/backend/shared/ssrf.ts`.
  - Shared SSRF validator:
    - enforces https-only
    - blocks userinfo (`https://user:pass@...`)
    - blocks common internal hostnames (`localhost`, `.local`, `.internal`, `.intra`)
    - blocks private/reserved IPv4 ranges and common IPv6 internal ranges
    - validates all resolved A/AAAA answers stay public (DNS rebinding safety check at validation time)
    - supports optional port allowlist (`allowedHttpsPorts`) when explicitly configured by server-side code
- Controller webhook endpoint validation now delegates to shared SSRF (no duplicate IP/DNS validator logic in controller):
  - Controller keeps legacy human-readable error messages by mapping typed SSRF error codes.
- Delivery-worker now uses shared building blocks:
  - URL validation delegates to shared SSRF before each delivery attempt.
  - HTTP delivery uses `cdktf/src/backend/shared/http.ts` (timeout + streaming response snippet capture).
  - Redirects are not followed (manual redirect policy); 3xx + Location remains a permanent failure.
  - Webhook payload JSON is serialized with `stableStringify()` for deterministic signing inputs.
- Time helpers are now centralized:
  - Removed ad-hoc `nowIso()/isoNow()/todayUTC()` definitions from lambdas and imported from `cdktf/src/backend/shared/time.ts`.

What changed (contracts + metrics)
- Authorizer metric cardinality hardening:
  - `auth.*_total` metrics no longer accept arbitrary exception messages as the `reason` dimension value; the value is now normalized to an allowlisted set (unknown -> `other`, unexpected failures -> `internal_error`).
- Admission Guard alarmability hardening:
  - Admission Guard now emits CloudWatch Embedded Metric Format (EMF) counters for decisions (`admission.decision_*`) while retaining the existing JSON marker used by any log-metric-filters.
- Agents ↔ infra auth contract alignment:
  - Python agents now parse bearer tokens by splitting on the FIRST dot so secrets may contain dots (matches infra authorizer parsing semantics).
  - Python agents treat environment as server-authoritative from the API key record; callers do not select env via headers/query.
  - Agent env-access gating now evaluates PROD-vs-SANDBOX decisions against the API key record environment (not caller input).
  - Agent EMF metrics no longer include `tenant_id` as a dimension (even when local DEBUG flags are set), to avoid accidental high-cardinality in CloudWatch.
- Workspace conventions:
  - Added repo-root `AGENTS.md` to document local build/test commands and the auth/env contract at a glance.
- Control-plane durability maturity:
  - Added a dedicated DynamoDB control-plane events table and controller writes one idempotent durable event record per admin/control-plane request (no secrets stored).
  - Added low-cardinality EMF metrics: `controlplane.event_write_total {env,outcome}` and `controlplane.event_write_error_total {env}`.

What changed (infra)
- WAF hardening behind deploy flags:
  - Added optional AWS managed rule groups:
    - `AWSManagedRulesAnonymousIpList` (flag: `WAF_ENABLE_ANONYMOUS_IP=1`)
    - `AWSManagedRulesBotControlRuleSet` (flag: `WAF_ENABLE_BOT_CONTROL=1`, inspection level via `WAF_BOT_CONTROL_INSPECTION_LEVEL`)
  - Managed rule groups continue to default to COUNT unless `WAF_MANAGED_RULES_BLOCK_MODE=1` is set at deploy time.
- Added authorizer throttles alarm (`AWS/Lambda Throttles` > 0) to surface edge/auth capacity issues quickly.
- Added key lifecycle widget to ops dashboard for quick rotation/revocation/next-secret usage visibility.
- Added a WAF rule activity widget (counted/blocked per rule name) to make managed-rule “count mode” visible during rollout.

Key lifecycle observability

  - `key.rotate_total {env,outcome}`
  - `key.revoke_total {env,outcome}`
- Authorizer already emits a low-cardinality "next-secret used" signal:
  - `kyc.api_key_next_secret_used_total {env}`
